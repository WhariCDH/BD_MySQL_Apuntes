/*******************************/
🔹 1. ¿Qué es una SUBCONSULTA?
/*******************************/
Es una consulta dentro de otra consulta.

👉 Sirve para:

Comparar valores
Filtrar resultados más complejos
Evitar hacer varios pasos manuales

💡 Idea mental:

Consulta principal → usa resultado de otra consulta

/***********************/
🔸 Tipos más importantes
/***********************/

✔ IN

Se usa cuando quieres comparar con una lista de resultados
👉 Ejemplo mental:
“Buscar empleados que están en ciertos departamentos”

✔ EXISTS

Verifica si existe al menos un resultado
👉 No le importa el valor, solo si hay datos
💡 Diferencia clave:
IN → compara valores
EXISTS → verifica existencia

/************************/
🔹 2. FUNCIONES DE FECHA
/************************/
Sirven para trabajar con tiempo y fechas reales

🔸 Principales

DAY() → día
MONTH() → mes
YEAR() → año
CURDATE() o GETDATE() → fecha actual
DATE_ADD() → sumar tiempo
DATEDIFF() → diferencia entre fechas

💡 Para qué se usan
-Calcular antigüedad
-Saber cuánto tiempo ha pasado
-Generar reportes por año/mes

👉 Ejemplo mental:
“¿Cuántos días lleva un empleado trabajando?”

/*************************/
🔹 3. FUNCIONES DE CADENA
/*************************/
Sirven para trabajar con texto (strings)

🔸 Principales

CONCAT() → unir textos
LENGTH() → longitud
SUBSTRING() → cortar texto
UPPER() → mayúsculas
LOWER() → minúsculas
TRIM() → eliminar espacios

💡 Para qué se usan
-Formatear nombres
-Limpiar datos
-Mostrar información más clara
👉 Ejemplo mental:
“Convertir nombre a MAYÚSCULAS o unir nombre + apellido”

/*****************************/
🔹 4. FUNCIONES DE CONVERSIÓN
/*****************************/
Sirven para cambiar el tipo de dato

🔸 Principales

CAST()
CONVERT()

💡 Para qué se usan
-Convertir número a texto
-Convertir fecha a string
-Ajustar datos para cálculos
👉 Ejemplo mental:
“Convertir salario decimal a entero”

/******************************/
🔥 5. IDEA CLAVE DE LA SEMANA
/******************************/
👉 Esta semana trata de hacer consultas más inteligentes y dinámicas

Antes (básico):

/*****************/
⚠️ ERRORES COMUNES

❌ Confundir IN con EXISTS
❌ No entender que la subconsulta se ejecuta primero
❌ Usar funciones sin entender el tipo de dato
❌ No manejar bien fechas (muy común)
